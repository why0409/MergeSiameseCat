# GEMINI.md: 《合成大暹罗》微信小游戏开发蓝图

## 一、 项目概述 (Project Overview)

* **游戏名称：** 《合成大暹罗》(Merge Siamese Cat)
* **游戏核心：** 2D 物理掉落 + 合成消除（类似“合成大西瓜”机制）。
* **目标体验：** 单手操作，随时开启/关闭，单局耗时 1~3 分钟，利用碎片时间释放压力。
* **技术栈：** Cocos Creator 3.8 (物理引擎) + TypeScript + 微信小游戏 API。
* **开发周期：** 单人开发，预计 3~5 天完成可上线原型。由于你有着扎实的软件开发背景，且 TypeScript 的面向对象、强类型特性与 C# 或 Java 非常相似，这部分脚本逻辑对你来说上手会极快。

## 二、 核心玩法与合成路线 (Core Gameplay & Merge Tree)

玩家通过点击屏幕顶部，释放不同大小的猫咪。两只体积和等级相同的猫咪在物理碰撞后，会合并成一只更高等级、体积更大的猫咪。

游戏的目标是合成出最顶级的“健康大暹罗”。当堆叠的猫咪超过屏幕顶部的警戒线时，游戏结束。

### 暹罗猫合成进化树 (10个阶段)

为了让合成过程更具期待感，我们将整个进化树设计为暹罗猫的成长史：

| 等级 (Level) | 形态设定 | 物理表现 (Radius) | 碰撞得分 |
| --- | --- | --- | --- |
| 1 | 逗猫羽毛球 | 极小，容易卡在缝隙中 | +1 |
| 2 | 毛线球 | 较小，弹性强 | +2 |
| 3 | 幼崽小暹罗 | 较小，重量轻 | +4 |
| 4 | 蓝眼小暹罗 | 中等偏小 | +8 |
| 5 | 调皮长腿暹罗 | 中等，开始占据一定空间 | +16 |
| 6 | 挖煤脸成年暹罗 | 中等偏大，具有一定重量 | +32 |
| 7 | 贵族长毛暹罗 | 大体积 | +64 |
| 8 | 霸气暹罗主子 | 极大体积，挤压下方空间 | +128 |
| 9 | 戴皇冠的暹罗王 | 极难达成，逼近警戒线 | +256 |
| 10 | **终极健康大暹罗** | 占据近半个屏幕，触发全屏特效 | +1000 |

## 三、 核心技术实现思路 (Technical Architecture)

这部分是整个项目的灵魂，我们把 90% 的运算交给引擎底层的物理系统。

### 1. 物理系统配置 (Cocos 2D Physics)

* **边界设定：** 在屏幕左、右、下三面设置静态的 `BoxCollider2D` 作为墙壁，防止猫咪掉出屏幕。
* **猫咪预制体 (Prefab)：**
* 挂载 `RigidBody2D`（刚体，类型设为 Dynamic，受重力影响）。
* 挂载 `CircleCollider2D`（圆形碰撞体，根据不同等级调整 Radius 半径）。
* 物理材质 (PhysicsMaterial)：设置适当的 `Friction`（摩擦力，防止无限滑动）和 `Restitution`（恢复系数/弹性，让猫咪掉落时有微弹的 Q 软感）。



### 2. 核心碰撞逻辑 (TypeScript)

只需要在猫咪的脚本中监听物理引擎的碰撞回调事件 `onBeginContact`：

* **判定：** 检查发生碰撞的两个节点是否具有相同的 `level` 属性。
* **销毁与生成：** 如果相同，销毁这两个节点，并在它们的中心坐标点实例化一个 `level + 1` 的新猫咪预制体。
* **防抖处理：** 物理引擎在同一帧可能会触发多次回调，需要加一个简单的布尔值状态锁，防止两只猫咪瞬间生成两个高级猫咪。

## 四、 商业化变现设计 (Monetization)

在不破坏极简心流的前提下，植入微信流量主广告。

1. **红线危机（核心收入）：**
* 触发条件：当猫咪停留超过屏幕顶部的警戒线 2 秒时，判定死亡。
* 弹出 UI：“主子堆不下了！🎬 看视频消除顶部 3 只猫咪并继续游戏”。


2. **底部 Banner (长尾收入)：**
* 在屏幕最下方 10% 的安全区域，常驻微信 Banner 广告。



## 五、 本地数据存储 (Local Storage)

因为没有复杂的成长系统，数据持久化极简：

* 使用微信 API `wx.setStorageSync('highestScore', score)` 保存历史最高分。
* 每次游戏结束时进行比对，若打破记录则播放特殊音效。
