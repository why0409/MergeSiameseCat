# GEMINI.md: 《合成大暹罗》微信小游戏开发蓝图 (Cocos 3.8.5)

## 一、 项目概述 (Project Overview)
* **游戏名称：** 《合成大暹罗》(Merge Siamese Cat)
* **游戏核心：** 2D 物理掉落 + 合成进化。
* **技术栈：** Cocos Creator 3.8.5 (LTS) + TypeScript + 微信小游戏 API。
* **当前状态：** ✅ 核心玩法闭环、✅ 视觉反馈美化、✅ 微信社交系统接入、✅ 引擎版本平滑升级。

## 二、 核心功能实现 (Feature Map)

### 1. 交互与反馈 (Juice & Feedback) ✅
* **丝滑跟随**：利用 Lerp 实现猫咪平滑物理跟随，消除瞬移感。
* **触感反馈**：集成微信 `vibrateShort`，在下落和合成时提供不同强度的物理震动。
* **连击系统**：实现 `Combo` 逻辑，多次连续合成触发动态 UI 提示。
* **动效增强**：接入 `Tween` 系统，实现分数的弹性跳动和猫咪合成的 BackOut 弹出动效。

### 2. UI 系统 (UI System) ✅
* **智能顶部栏**：使用 Widget 适配不同屏屏，包含当前得分、历史最高分。
* **结算面板**：自定义 GameOver 遮罩层，支持最终分数展示与排行榜跳转。
* **预警系统**：Deadline 线在危险时自动闪烁（呼吸灯效果），并在失败时变红常亮。

### 3. 微信集成 (Social Features) ✅
* **云存储同步**：自动上传 `score` 至微信云端 KV 数据库。
* **排行榜桥接**：主域 `WeChatRank` 脚本通过 `SubContextView` 与开放数据域通信。

## 四、 维护与稳定性说明
- **物理系统注意事项**：当前项目已升级至 3.8.5 LTS。物理引擎使用 Box2D。
    - **状态锁定处理**：严禁在碰撞回调 (`onBeginContact`) 中直接修改 Node 的 `active`、`scale` 或 RigidBody 的 `enabled` 状态。
    - **视觉消除方案**：合成时采用 `setScale(0)` + `setWorldPosition(9999, 9999)` 的即时方案，并在 `scheduleOnce` (下一帧) 执行最终的 `destroy()`。
- **UI 层级管理**：
    - **层级竞争问题**：由于 `GameOverPanel` 在显示时会强制 `setSiblingIndex` 置顶，调用 `WeChatRank` 显示排行榜时必须同步执行 `rankPanel.setSiblingIndex` 操作，否则排行榜会被结算面板遮挡（表现为点击无反应）。
- **逻辑唯一性**：统一使用根节点的 `uuid` 进行排序，确保双向碰撞只触发单次合成逻辑。
